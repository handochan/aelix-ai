# ADR-0182 — #21 W2: manifest `contributes.tui_widgets` adapter

- **Status:** Accepted — **LIVE**.
- **Date:** 2026-07-04
- **Sprint:** Declarative extension surface, wave 2 (#21 remainder — the tui_widgets family, recon'd as the lowest-cost of the three deferred families).
- **Pi pin:** `earendil-works/pi@734e08e`. **aelix-original** — pi has no capability manifest (ADR-0181); the *imperative* widget surface (`setWidget`, `WidgetFactory`, `WidgetPlacement`) is pi-parity (types.ts:97-167), and this adapter is a thin declarative front for it.
- **Relates:** ADR-0181 (W1: activation engine + the get_shortcuts/live-read idioms this follows), ADR-0096 (manifest v1 schema — `TuiWidgetContrib` existed, Pydantic-validated then dropped), ADR-0105 (AelixTUIContext; per-extension `ui_tui_trusted` gating deferred there — this ADR ships its first declarative consumer), ADR-0102 (Tier-4b hooks trust-gate precedent). GitHub #21 (partial — themes/descriptors remain).

## Context

`contributes.tui_widgets` (`TuiWidgetContrib {slot, factory}`, manifest.py:129-132) had ZERO consumers — declared widgets were validated and dropped. The imperative path is complete: `ctx.ui.set_widget(key, content, options)` → `AelixTUIContext.set_widget` (context.py:837) invokes a `WidgetFactory = Callable[[TUI, Theme], Component]` ONCE, renders `Component.render(width) -> list[str]` at width 80, and pushes static raw-ANSI lines into the chrome's placement dict (`chrome.set_widget`, last-wins by key, live-safe pre- and post-run). `contrib.factory` is a `module:attr` import spec — the SAME colon-form the loader already resolves (`_factory_from_module`), and resolving it EXECUTES plugin code, so it must never happen in a metadata-only scan (ADR-0181 Slice A) nor for a pending lazy shell.

## Decision

- **Adapter** (`tui/ext_widgets.py`, new): `apply_manifest_widgets(runner, ui, applied, *, pending)` reconciles the chrome against the CURRENT extension set — for each loaded extension's `manifest.contributes.tui_widgets`: `slot` must be a `WidgetPlacement` literal (`above_editor`/`below_editor`; unknown → warn + skip, forward-compatible), the chrome key is namespaced `ext:{name}:tui_widgets[{i}]` (extensions cannot clobber each other or built-ins like `__stream__`), `factory` resolves via `_factory_from_module` and is passed to `ui.set_widget` as a `WidgetFactory` (the imperative contract — static-snapshot semantics shared with that path). Keys previously applied but no longer declared are un-painted. Per-contrib never-raises (warn + skip); `runner` duck-typed.
- **Wiring** (`shell.py`): `_apply_ext_widgets(harness)` runs inside `_rebind` right after `bind_ui` — so startup, `/resume`·`/fork` swaps, and #24 `/reload` all re-apply against the new harness's extension set (a removed plugin's widgets un-paint; the factory re-renders, so content refreshes per rebind). Reads the PASSED harness, not `runtime_host.harness` (no mid-swap staleness). Wired via `context.set_widget`, NOT a chrome-constructor kwarg — so the injected-chrome test path exercises it too. TUI-only by construction (the only caller is `run_tui`; the headless `ctx.ui` stub raises on `set_widget`).
- **Eager forcing** (`loader._is_lazy_eligible`): `contributes.tui_widgets` now disqualifies deferral, joining `contributes.tools`/`hooks` — the adapter only reads LOADED extensions and there is no post-activation re-apply seam, so a deferred plugin's declared widgets would silently never paint (the same silent-vanish class as the W1 tools MEDIUM). The adapter also carries a defensive `pending` fence (never import a pending shell's factory).
- **Trust gate** (`loader._resolve_factory`, manifest branch): `[[contributes.tui_widgets]]` requires `capabilities.ui_tui_trusted = true`, mirroring the Tier-4b hooks/`shell_exec` gate — a declared widget's factory is plugin code executed in the user's terminal. Fires BEFORE `_factory_from_module` imports the entry module, so a denied plugin executes NO code at all (data before code — the review's MEDIUM: the gate originally sat in `_invoke_factory`, after the import). A defensive second fence stays in `_invoke_factory` for direct-factory callers that thread a manifest without entry resolution. Failure is a per-extension error collected into `loaded.errors` (other plugins keep loading); the gate also guards the lazy-activation path (`activate_pending_extension` → `_resolve_factory`) should eligibility ever regress. **Scope honesty (review LOW):** capabilities are self-declared and the *imperative* `ctx.ui.set_widget` is ungated (any loaded extension may call it) — this gate is a declarative-surface consistency check ("what runs in your terminal is declared"), not a code-execution boundary; per-extension UI gating remains deferred as in ADR-0105.

## Consequences

- A manifest can now paint TUI widgets with zero imperative code; `slot` doubles as the placement selector. Richer needs (dynamic content, removal, custom keys) stay on the imperative `ctx.ui.set_widget`.
- Static-snapshot semantics inherited from the imperative path: the factory renders once per apply (re-renders on every rebind, not per frame).
- Reconcile un-paints on removal AND on placement change (the chrome keeps one dict per placement — review MEDIUM, live-reproduced permanent leak); same-name extensions collide on the widget key → first wins with a warning (the `get_shortcuts` convention).
- **Documented asymmetry (review LOW):** manifest widgets reconcile away on rebind/reload, but *imperative* `ctx.ui.set_widget` widgets from a removed or rollback-failed plugin persist until TUI restart — chrome mutations are UI side effects outside the activation rollback's 8 Extension buckets (pre-existing, ADR-0181).
- `_RENDER_WIDTH = 80` best-effort width applies (same as imperative factories).
- #21 remainder after this: `contributes.{themes,descriptors}` (greenfield / shape-mismatch — descriptors need a schema decision), lazy `on_tool_call`, manifest-MCP on `/reload`. `docs/guides/extension-authoring.md` still documents no manifest features at all (pre-existing gap, owner-visible scope decision).
- **Gate:** pytest 4594 pass / 1 skip (+16 vs the post-W1 4578 baseline) · ruff clean · pyright 8 pre-existing `scripts/pyright_spike.py` errors only.
